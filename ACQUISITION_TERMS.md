# VeritAI News Ecosystem: Acquisition Terms

This document outlines the terms for the Full IP Acquisition of the VeritAI News ecosystem.

### 1. Assets Included
The acquisition includes the total transfer of:
- The `veritai-news` GitHub Organization.
- All associated repositories: Including but not limited to `veritai-api`, `veritai-mobile`, and `veritai-dashboard`.
- All Intellectual Property, source code, copyrights, and branding associated with the "VeritAI News" ecosystem.

### 2. Prior Licenses (Grandfathering)
The Buyer acknowledges that the VeritAI News ecosystem has been sold as a "Source-Available" product. The Buyer irrevocably agrees to honor all Standard commercial licenses granted via GitHub Sponsors prior to the official Transfer Date. 

**Scope of Grandfathering:** "Honoring" these licenses strictly means the Buyer recognizes the existing users' perpetual right to use the software versions available up to the Transfer Date. The Buyer may not revoke these licenses nor initiate copyright "takedown" requests against these authorized users. **However, the Buyer assumes absolutely no obligation to provide future updates, maintenance, or technical support to these grandfathered users.**

### 3. Post-Acquisition Control & Privatization
The Buyer acquires absolute control over the future of the codebase. Immediately following the transfer of the GitHub Organization, the Buyer has the explicit right to:
- Transition all previously public repositories to **private** status.
- Permanently **close-source** the project.
- Cease all future public distribution, sales, or licensing of the VeritAI ecosystem. 
*(Existing $99 Licensees retain the right to the code they have already downloaded/forked, but will lose access to the official repositories once made private by the Buyer).*

### 4. Handover Process
1. **Verification:** The potential Buyer emails `fulleni@hotmail.com` to confirm availability and intent to acquire.
2. **Payment:** The Buyer completes the $5000 Full IP Acquisition sponsorship via GitHub Sponsors.
3. **Agreement:** The Seller provides a formal Intellectual Property Assignment Agreement for digital signature.
4. **Transfer:** Upon signature, the Seller initiates the "Transfer Ownership" process for the GitHub Organization to the Buyer's designated GitHub account.

### 5. No Ongoing Obligation
Upon successful transfer of the GitHub Organization, the original developer (Seller) is completely released from all maintenance, support, and update obligations. The project and its associated IP are acquired "As-Is."
